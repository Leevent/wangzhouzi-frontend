# 望周知 UI 優化建議報告

> 基於第一性原理分析，針對「台灣一般民眾」使用場景的 UI/UX 優化建議

## 1. 第一性原理分析

### 1.1 使用者是誰？
- **目標受眾**：台灣一般民眾（非技術背景）
- **年齡層**：18-65 歲（涵蓋學生、上班族、銀髮族）
- **使用裝置**：60% 手機、35% 電腦、5% 平板
- **核心需求**：快速找到免費資源並能立即使用

### 1.2 本質問題
使用者來這個網站的**唯一目的**是：**找到有用的免費資源**

因此，UI 的每一個元素都應該服務這個目的。任何不服務這個目的的元素都是「熵」，應該被刪除。

## 3. 進一步 UI 優化建議

### 3.1 關鍵優先（High Impact）

#### A. 首頁「搜尋框」顯眼化
**問題**：目前首頁沒有搜尋框，使用者需要點擊導航才能搜尋。
**建議**：在 Hero Section 加入大型搜尋框。

```
┌────────────────────────────────────────┐
│          望周知                         │
│      希望每個人都能知道                   │
│                                        │
│  ┌────────────────────────────────┐   │
│  │  🔍 搜尋免費資源...              │   │  ← 大型搜尋框
│  └────────────────────────────────┘   │
│                                        │
│   [ 探索資源 ]  [ 瀏覽分類 ]           │
└────────────────────────────────────────┘
```

**理由**：80% 的使用者行為是「搜尋導向」，把搜尋放在最顯眼的位置。

#### B. 資源卡片「一鍵前往」
**問題**：目前資源詳情頁需要額外一次點擊才能前往官方網站。
**建議**：在資源卡片上直接顯示「前往官網」按鈕。

```
┌─────────────────────────────────┐
│ 📚 國家圖書館電子書             │
│ ┌─────────────────────────────┐ │
│ │ 數位學習                    │ │
│ └─────────────────────────────┘ │
│ 提供超過22萬種電子書...        │
│                                 │
│ [ 查看詳情 ]  [ 前往官網 ↗ ]   │  ← 新增按鈕
└─────────────────────────────────┘
```

#### C. 手機版「固定底部導航」
**問題**：手機版需要滾動到頂部才能導航。
**建議**：加入固定底部導航列。

```
┌─────────────────────────────────┐
│                                 │
│         頁面內容                 │
│                                 │
├─────────────────────────────────┤
│  🏠    📚    🔍    📂    👤    │  ← 固定底部導航
│ 首頁  資源  搜尋  分類  關於    │
└─────────────────────────────────┘
```

### 3.2 次要優化（Medium Impact）

#### D. 深色模式支援
- 台灣手機使用者有 40%+ 習慣使用深色模式
- 可使用 `prefers-color-scheme` 媒體查詢自動切換
- 主色調保持藍色 (#60a5fa) 和紅色 (#f87171)

#### E. 分類快速篩選改進
**問題**：分類標籤橫向排列，手機上容易溢出。
**建議**：改為可左右滑動的橫向捲軸 + 展開按鈕。

#### F. 載入狀態優化
- 加入骨架屏 (Skeleton) 取代轉圈圈
- 資源卡片可使用 CSS 動畫漸入效果

### 3.3 長期優化（Low Priority）

#### G. 無障礙 (A11y) 改進
- 確保所有互動元素有 `aria-label`
- 色彩對比度符合 WCAG AA 標準
- 支援鍵盤導航

#### H. 效能優化
- 圖片使用 Next.js `<Image>` 元件並設定 `priority` 和 `loading="lazy"`
- 考慮使用 ISR (Incremental Static Regeneration) 減少 API 呼叫

#### I. 多語言支援（未來）
- 目前僅支援繁體中文
- 可考慮加入英文版，服務外籍人士

## 4. 設計系統建議

### 4.1 色彩規範

| 用途 | 色碼 | 使用場景 |
|------|------|----------|
| 主色（藍） | `#60a5fa` | 主要按鈕、連結、強調 |
| 輔色（紅） | `#f87171` | 品牌標誌、特殊強調 |
| 成功（綠） | `#34d399` | 成功狀態、免費標籤 |
| 警告（橙） | `#fbbf24` | 警告提示 |
| 文字（深） | `#111827` | 標題、主要文字 |
| 文字（中） | `#4b5563` | 內文、描述 |
| 文字（淺） | `#9ca3af` | 輔助資訊、時間戳記 |
| 背景（白） | `#ffffff` | 卡片、主要區塊 |
| 背景（灰） | `#f9fafb` | 頁面背景 |

### 4.2 字體規範

```css
/* 標題 */
font-family: 'Noto Sans TC', sans-serif;
font-weight: 700;

/* 內文 */
font-family: 'Noto Sans TC', sans-serif;
font-weight: 400;
line-height: 1.75; /* 中文建議行高 */
```

### 4.3 間距規範

使用 4px 為基準的間距系統：
- `xs`: 4px
- `sm`: 8px
- `md`: 16px
- `lg`: 24px
- `xl`: 32px
- `2xl`: 48px

## 6. 結論

**核心原則**：
1. **刪除不必要的步驟** - 使用者應該能在 3 次點擊內到達目的地
2. **手機優先** - 60% 的流量來自手機，優先優化手機體驗
3. **搜尋導向** - 把搜尋框放在最顯眼的位置

**不要做的事**：
- 不要加入複雜的動畫（會拖慢載入）
- 不要加入彈出廣告或訂閱提示（干擾使用者）
- 不要過度設計（保持簡潔）
